# 我是破烂王 - 产品需求文档 (PRD)

## 1. 产品概述

| 属性 | 说明 |
|------|------|
| **游戏名称** | 我是破烂王 |
| **游戏类型** | 单机合成+经营类小游戏 |
| **核心玩法** | 玩家通过合成破烂提升价值，向收购商售卖获取金钱，再用金钱购买盲盒（垃圾桶）获得更多破烂，不断循环 |
| **目标用户** | 休闲游戏玩家，喜爱合成、收集和随机性乐趣的用户 |
| **技术实现** | HTML5小游戏，不支持本地存储进度 |

## 2. 游戏目标

无明确终点，玩家自由体验合成与售卖循环，追求高价值破烂或收集稀有词条。

## 3. 核心规则

### 3.1 初始状态

- 游戏开始时，玩家拥有 **4件随机破烂**（破袜子、易拉罐、苹果核、旧手机）
- 每件破烂以文字卡牌形式呈现，卡牌包含以下属性：
  - **合成次数**：初始为0

### 3.2 合成操作

| 项目 | 说明 |
|------|------|
| **条件** | 玩家从背包中选择 **两件不同的破烂的卡牌**（可以是相同名字但不同实例），且两者的合成次数均 ≤4（即尚未达到5次上限） |
| **操作** | 将一张卡牌拖拽到另一张卡牌上 |
| **结果** | 1. 消耗选中的两件破烂卡牌<br>2. 在原地弹出一张新破烂卡牌<br>3. 新破烂自动加入玩家背包 |
| **新卡牌属性** | - **名字**：由AI根据两件旧破烂的名字和词条动态生成<br>- **合成次数** = max(旧破烂A的合成次数, 旧破烂B的合成次数) + 1 |
| **限制** | 若任意一件破烂的合成次数已为5，则"合成"按钮禁用并提示"该破烂已满级，无法合成，请找收购商出售" |

### 3.3 次数上限

- 每件破烂卡牌的合成次数 **最大为5**
- 当一件破烂的合成次数达到5时：
  - 不可再作为合成材料（拖拽到另一卡牌上时会自动弹开）
  - 只能通过"向收购商出售"处理
- 合成次数显示在文字卡牌的右下角（如"⚙️3/5"）

### 3.4 向收购商售卖

- 玩家可点击主页面上的"收购商"交互按钮，进入售卖页面
- 玩家拖拽任意卡牌进入售卖页面卡槽，点击 "售卖"按钮
- 立即获得与该破烂 **售价等值的金币**（由AI，即收购商根据设定估价）
- 该破烂卡牌从背包中消失
- **售卖无次数限制**，随时可进行

### 3.5 购买盲盒（垃圾桶）

| 属性 | 说明 |
|------|------|
| **位置** | 主界面上，一个"垃圾桶"的交互按钮 |
| **价格** | 固定 **20金币**（可调，初始建议值） |
| **操作** | 点击"垃圾桶"交互按钮，若玩家金钱 ≥ 20，则扣除20金币 |
| **产出** | 随机获得 **0~5个随机垃圾**（每个垃圾也是一张独立的文字卡牌） |
| **数量概率** | 0个、1个、2个、3个、4个、5个概率相等（各约16.7%） |
| **垃圾属性** | - 合成次数 = 0<br>- 名字：从 **扩展垃圾词库** 中随机选取，不能是基础词库中的四个名字（即不是破袜子、易拉罐、苹果核、旧手机），但可以重复获得相同名字 |

**扩展垃圾词库示例**：
> 烂菜叶、碎玻璃、塑料袋、废电池、旧报纸、易拉环、瓶盖、吸管、奶茶杯、烟头、破布条、断梳子、假牙、香蕉皮、放屁垫、毛线团、狗骨头、沃尔玛购物袋、邮票、弹簧、蟑螂、老鼠尸体、刀柄、断剑、鸡蛋壳、泡面叉子、假发、单只拖鞋、空打火机、遥控器等

## 4. 经济系统

| 项目 | 说明 |
|------|------|
| **初始金钱** | 20金币（v1.2修改：从0改为20，让玩家可以立即体验盲盒功能） |
| **金钱来源** | 售卖破烂 |
| **金钱消耗** | 购买破烂盲盒（20金币/个） |
| **价格规则** | - 初始破烂（开局的4个破烂）售价：1~3金币随机<br>- 合成后的破烂售价由收购商（即AI）确认<br>- 无其他价格修正 |

## 5. 界面与交互需求

### 5.1 主界面布局

```
┌─────────────────────────────────────────┐
│  顶部栏：显示当前金币数量                    │
├─────────────────────────────────────────┤
│                                         │
│  合成区域（顶部栏和底部栏之间）              │
│  ┌─────────┐         ┌─────────┐       │
│  │ 垃圾桶  │         │ 收购商  │       │
│  │ (20金币)│         │ (售卖)  │       │
│  └─────────┘         └─────────┘       │
│                                         │
│  提示信息区（合成结果预览或错误提示）         │
│                                         │
├─────────────────────────────────────────┤
│  底部栏（背包区域）：横向网格卡槽展示所有卡牌   │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ...      │
│  │名字│ │名字│ │名字│ │名字│            │
│  │⚙️0/5│ │⚙️2/5│ │⚙️MAX│ │⚙️1/5│            │
│  └────┘ └────┘ └────┘ └────┘            │
└─────────────────────────────────────────┘
```

**卡牌内容布局**（从上到下或从左到右）：
- 第一行：名字
- 第二行：⚙️合成次数（例如"⚙️2/5"）

**满级卡牌特殊样式**：
- 合成次数显示为"⚙️MAX"或"⚙️5/5"
- "合成选择"按钮置灰或不可见
- 卡牌加上动态流光边框

### 5.2 交互流程

#### 1. 初始加载
- 生成4件初始破烂文字卡牌，显示在背包

#### 2. 合成操作
- 从背包区域卡槽拖拽第一张卡牌进入合成区域，该卡牌从背包消失，显示在合成槽中
- 拖拽第二张卡牌进入合成区域，两张卡牌自动消耗并生成新卡牌加入背包
- 若点击合成槽的✕按钮，已拖入的卡牌返回背包
- 若任一卡牌满5次则禁用并提示
- 若新破烂合成次数达到5，卡牌样式变为满级样式

#### 3. 售卖流程
1. 点击"收购商"交互按钮，出现售卖界面弹窗
2. 弹窗具备单张卡片卡槽和"售卖"按钮、以及收购商立绘
3. 玩家从背包区域拖入任意一张卡牌进入弹窗卡槽
4. 收购商会根据玩家拖入卡槽的卡牌进行估价（同时说明心情，心情会影响估价）
5. 以一侧的气泡框内的文字形式呈现估价
6. 玩家点击售卖按钮后售卖成功
7. 售卖后金币增加，该卡牌从背包移除

#### 4. 购买盲盒
1. 点击"垃圾桶"
2. 检查金币，不足则提示
3. 足够则扣除金币，随机生成0~5张垃圾文字卡牌
4. 依次加入背包（逐个动画）
5. 若产出0个，提示"鬼运气！什么都没捞到！"

### 5.3 异常与限制

| 场景 | 处理 |
|------|------|
| 背包为空 | 无法进行任何合成，需购买盲盒获取垃圾 |
| 合成时两件相同实例 | 不可选（需两件不同槽位） |
| 满次数卡牌 | "合成选择"按钮置灰或不可点击 |
| 游戏结束条件 | 若玩家背包卡牌数量≤1且金币数量＜20，则游戏结束 |
| 游戏结束处理 | 显示"游戏结束"文字，3秒后自动回到初始界面 |

## 6. 非功能性需求

| 项目 | 说明 |
|------|------|
| **界面风格** | 垃圾堆风格，废土/涂鸦美术，所有破烂仅以文字卡牌呈现，无图片素材，降低美术成本 |
| **卡牌样式** | 米色背景，黑色文字，边缘磨损效果，字体清晰 |
| **音效** | 售卖硬币声、合成叮当声、翻动垃圾声 |
| **性能** | 支持20张以上卡牌流畅滚动 |

## 7. 边界情况说明

- 盲盒产出垃圾的合成次数均为0，可继续参与合成
- 游戏无回合限制，可无限游玩

## 8. 技术实现方案与优先级

### 8.1 技术栈与运行环境

| 项目 | 说明 |
|------|------|
| **平台** | Trae |
| **前端技术** | 纯HTML5 + CSS3 + 原生JavaScript（ES6），无第三方依赖 |
| **存储方式** | 不支持本地存储，游戏状态仅保存在内存中（刷新页面即重置） |
| **渲染方式** | DOM元素模拟卡牌拖拽（便于实现拖拽、碰撞检测和动态生成） |
| **AI接口** | 预留HTTP API调用位置，支持对接外部大模型服务 |

### 8.2 模块划分与优先级

#### P0（必须实现，核心可玩）

| 模块 | 说明 |
|------|------|
| **1. 基础数据模型** | - 定义Card对象：`{ id, name, level, combineCount, value? }`<br>- 注意PRD中售价由收购商AI估价，故本地不存售价，仅在售卖时请求AI<br>- 初始卡牌生成：从基础词库随机取4个名字，combineCount=0<br>- 扩展垃圾词库用于盲盒 |
| **2. 背包网格布局与拖拽合成** | - 底部栏横向卡槽（flex布局），支持拖拽（drag & drop）两张卡牌到合成区域<br>- 第一张卡牌拖入合成区域后从背包消失，显示在合成槽中，可点击✕按钮取消<br>- 第二张卡牌拖入后触发合成逻辑：校验条件（不同卡牌、combineCount≤4），计算新combineCount = max+1，调用AI获取新卡牌名字<br>- 生成新卡牌并加入背包，删除旧卡牌，合成完成后检查游戏结束 |
| **3. 收购商售卖弹窗** | - 点击"收购商"按钮弹出模态框，包含一个卡槽（可拖拽放入卡牌）、售卖按钮、收购商立绘（可用emoji或文字代替）、气泡框显示估价和心情<br>- 拖拽卡牌进入弹窗卡槽时，调用AI估价接口（传入卡牌名字、合成次数），返回金币数和心情文字<br>- 点击售卖后增加金币，删除卡牌，关闭弹窗（或继续售卖） |
| **4. 垃圾桶盲盒购买** | - 点击"垃圾桶"按钮，检测金币≥20，扣除后随机0-5，从扩展词库随机取名字生成新卡牌（combineCount=0），逐个动画加入背包<br>- 金币不足提示 |
| **5. 游戏结束检测** | - 每次售卖、合成、购买盲盒后，检查条件：背包卡牌数量≤1 且 当前金币<20 → 显示"游戏结束"文字，3秒后重置到初始状态（4张初始卡牌，金币0） |

#### P1（增强体验，第二优先级）

| 模块 | 说明 |
|------|------|
| **1. 满级卡牌流光边框与禁用拖拽** | combineCount=5的卡牌添加CSS动画边框，并设置draggable=false，使其无法被拖拽合成。若尝试拖拽到合成区域，提示"已满级，请出售" |
| **2. 合成动画与音效** | 卡牌接触后播放"叮"声（Web Audio），新卡牌从合成区域弹入背包的飞入动画。售卖硬币声，盲盒翻动声 |
| **3. AI调用缓存与降级** | 合成时AI生成名字可能较慢，增加loading状态，若超时或失败则使用规则生成兜底名字（如"合成物A"+随机数字）。估价接口同样增加超时降级（按合成次数*2+随机值估价） |
| **4. 拖拽视觉反馈** | 拖拽中卡牌半透明，合成区域高亮，两张卡牌重叠时闪烁提示可合成 |

#### P2（优化与拓展，后续迭代）

| 模块 | 说明 |
|------|------|
| **1. 卡牌数量上限管理与滚动条** | 当卡牌超过20张时，背包区域出现横向滚动条，支持触摸滑动 |
| **2. 收购商心情动态** | 心情影响估价的波动因子（如±20%），增加趣味性（需AI接口支持心情输出） |
| **3. 声音开关与主题切换** | 添加静音按钮，浅色/深色主题 |

### 8.3 关键实现细节（供开发参考）

#### 8.3.1 拖拽合成实现（不使用HTML5原生drag，避免样式受限）

- 使用鼠标事件（mousedown/mousemove/mouseup）模拟拖拽，生成克隆元素跟随鼠标
- 合成区域作为"放置目标"，检测鼠标松开时克隆元素与合成区域是否重叠，以及当前是否已有第一张卡牌被"暂存"
- 简化方案：点击卡牌时高亮，再点击另一张卡牌时触发合成（避免复杂拖拽）。但PRD明确要求"将一张卡牌拖拽到另一张卡牌上"，建议采用上述自定义拖拽

#### 8.3.2 游戏状态管理

```javascript
// 全局对象
gameState = {
  gold: 0,
  cards: []
}
// 每次修改状态后重新渲染背包（部分框架可做虚拟DOM，简单项目中直接整体刷新背包区域，保持逻辑清晰）
```

#### 8.3.3 游戏结束重置

- 重置函数重置gold和cards，并重新生成4张初始卡牌
- 3秒计时器期间禁用所有操作按钮

## 9. AI接口设计（预留对接）

### 9.1 接口通用约定

| 属性 | 说明 |
|------|------|
| **请求方式** | HTTP POST，JSON格式 |
| **超时时间** | 3秒 |
| **错误处理** | 若接口不可用或超时，使用兜底规则生成结果，并提示"收购商走神了，勉强估了个价" |

### 9.2 合成命名接口

**端点**：`/api/synthesize`（可配置）

**请求体**：
```json
{
  "card1_name": "破袜子",
  "card2_name": "易拉罐",
  "card1_level": 2,
  "card2_level": 1
}
```

**响应体**：
```json
{
  "new_name": "金属补丁",
  "success": true
}
```

**兜底规则**：返回合成物 + (card1_name.slice(0,2) + card2_name.slice(0,2))，例如"破袜易拉"

### 9.3 收购商估价接口

**端点**：`/api/evaluate`

**请求体**：
```json
{
  "card_name": "生锈的易拉罐",
  "combine_count": 3,
  "mood": "happy"
}
```

**响应体**：
```json
{
  "price": 12,
  "mood_text": "今天心情不错，给你个好价钱！",
  "success": true
}
```

**兜底规则**：
- 价格 = (combine_count + 1) * 2 + 随机(-2~2)
- 心情文字为"老板随便估的"

### 9.4 本地Mock模式（开发调试）

在Trae环境中，若未配置真实AI接口，前端可启用Mock模式：
- **合成命名**：从预设组合表中随机返回（简单映射规则）
- **估价**：返回 (card.combine_count + 1) * 3 + 随机0-5
- 正式环境通过环境变量切换至真实API地址

### 9.5 接口配置文件示例

```javascript
const AI_CONFIG = {
  baseURL: "https://your-ai-service.com/api",  // 预留替换
  endpoints: {
    synthesize: "/synthesize",
    evaluate: "/evaluate"
  },

```

## 10. 需求变更记录

### v1.1 (2026-05-23)

#### 变更1：合成区域卡牌显示逻辑优化
**变更内容：**
- 拖拽卡牌到合成区域后，该卡牌立即从背包区域消失
- 合成区域显示已拖入的卡牌，并显示✕按钮用于取消合成
- 点击✕按钮后，卡牌返回背包区域

**变更原因：**
- 避免玩家混淆，明确显示哪些卡牌正在参与合成
- 防止同一卡牌被重复用于多个合成操作

**影响范围：**
- `js/cards.js` - `renderBackpack()`、`handleCardDropOnSynthesis()`、`returnCardToBackpack()`
- `index.html` - 合成槽UI

#### 变更2：游戏结束判定时机调整
**变更内容：**
- 游戏结束判定仅在以下操作完成后进行：
  - 合成操作完成后（两张卡牌消耗，新卡牌生成）
  - 售卖操作完成后
  - 购买盲盒操作完成后
- 合成过程中（消耗两张旧卡牌时）不触发游戏结束判定

**变更原因：**
- 避免合成过程中因临时卡牌数量减少而误判游戏结束
- 确保游戏结束判定基于稳定的游戏状态

**影响范围：**
- `js/state.js` - `addCard()`、`addCards()`、`removeCard()` 添加 `checkEnd` 参数
- `js/synthesis.js` - 合成完成后统一检查游戏结束

### v1.2 (2026-05-23)

#### 变更3：合成区域宽度调整
**变更内容：**
- 合成区域最小宽度从200px增加到400px

- 提供更好的视觉平衡和用户体验
- 为合成槽提供更多展示空间

**影响范围：**

#### 变更4：收购商弹窗拖拽修复
**变更内容：**
- 修复收购商弹窗打开后无法拖拽卡牌到弹窗槽位的问题
